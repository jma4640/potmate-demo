# 야끼테이 (やきてい) 키오스크 웹 설계

## 1. 개요

이 프로젝트는 `야끼테이 (やきてい)` 매장의 야끼소바 주문을 위한 정적 키오스크 웹이다.  
기존 프로젝트는 빌드 도구 없는 `HTML/CSS/JS` 구조이므로, 이번 버전도 같은 형태를 유지한다.

핵심 목표는 다음과 같다.

- 따뜻한 이자카야 무드의 브랜드형 키오스크 화면 제공
- 실제 음식 사진 중심의 빠른 주문 경험 제공
- `기본 인기 조합 + 현재 세션 누적 데이터` 기반의 실시간 토핑 추천 제공
- 여러 메뉴를 장바구니에 담고 메뉴별로 개별 취소할 수 있는 주문 경험 제공
- 실제 결제 연동 없이 주문 완료 화면까지 동작하는 프론트엔드 데모 제공

이번 수정 버전은 브라우저 내에서 동작하는 프론트엔드 중심 구현이며, 추천 데이터는 `localStorage`를 사용해 유지한다. 장바구니는 여러 개의 메인 메뉴 라인 아이템을 담을 수 있는 멀티카트 구조로 설계한다.

## 3. 브랜드 및 시각 방향

### 컬러 시스템

- 배경 베이스: `#FFF8F0`
- 메인 포인트: `#C08552`
- 서브 포인트: `#8C5A3C`
- 강한 텍스트/딥 브라운: `#4B2E2B`

### 시각 컨셉

- 크림 톤 배경 위에 카라멜 브라운 포인트를 사용하는 따뜻한 이자카야 무드
- 실제 음식 사진이 가장 먼저 눈에 들어오는 프리미엄 키오스크 레이아웃
- 카드, 패널, 버튼, 합계 영역이 떠 보이도록 깊은 그림자와 하이라이트를 사용하는 소프트 3D 스타일
- 평면적인 단색 배경 대신 그라데이션, 유리판 느낌의 패널, 겹치는 레이어로 입체감 형성
- 작은 화면에서는 세로 스택, 큰 화면에서는 좌측 메뉴/우측 주문 영역의 2단 구조 유지

### 이미지 전략

- 야끼소바 사진은 공개 사용 가능한 실제 사진을 사용한다
- 구현 시점에 재사용 가능한 출처에서 확보한 이미지를 프로젝트 내부 로컬 에셋으로 저장한다
- 외부 이미지 URL에만 의존하지 않고, 로컬 에셋 기준으로 안정적으로 렌더링한다
- 사진이 누락되거나 로드 실패해도 카드가 깨지지 않도록 폴백 배경과 대체 텍스트를 함께 설계한다
- 추후 실제 매장 사진으로 교체할 수 있도록 메뉴 데이터와 이미지 경로를 분리한다

## 4. 정보 구조 및 화면 구성

### 전체 구조

메인 화면은 `2단 레이아웃`으로 구성한다.

- 좌측: 메뉴 갤러리
- 우측 상단: 현재 선택 메뉴 커스터마이징 패널
- 우측 하단: 장바구니 및 총액 패널

### 상단 헤더

- 브랜드명 `야끼테이 (やきてい)`
- 짧은 소개 문구
- 실시간 추천 또는 인기 배지
- 3D 스타일을 강조하는 배경 장식과 유광 포인트

### 좌측 메뉴 갤러리

- `기본 야끼소바`, `소금 야끼소바`를 실제 사진과 함께 가장 크게 노출
- 각 카드에는 음식 사진, 메뉴명, 짧은 설명, 가격, 대표 배지를 포함
- 카드 자체가 살짝 떠 있는 듯한 깊이감과 반사광을 가진다
- 음료는 별도 섹션으로 분리하고 시각적 우선순위는 메인 메뉴보다 낮춘다
- 메뉴 카드를 누르면 우측 상단 커스터마이징 패널이 해당 메뉴 기준으로 갱신된다

### 우측 상단 커스터마이징 패널

현재 선택한 메뉴를 세부 조정하는 영역이다.

- 선택한 메인 메뉴의 썸네일, 이름, 가격
- 실시간 인기 토핑 조합 추천 카드
- 토핑 선택 영역
- 음료 선택 영역
- 메인 메뉴 수량 조절 영역
- 현재 구성 소계
- `장바구니 담기` 버튼

### 우측 하단 장바구니 패널

장바구니에는 사용자가 담은 메뉴 세트들이 라인 아이템으로 쌓인다.

- 메뉴명
- 선택 토핑 목록
- 선택 음료 목록
- 메뉴 수량
- 라인 금액
- 메뉴별 삭제 버튼
- 장바구니 총액
- `주문하기`, `전체 비우기` 버튼

### 완료 상태

`주문하기`를 누르면 실제 결제 대신 주문 완료 화면 또는 완료 모달을 보여준다.

- 장바구니에 담긴 전체 메뉴 세트 요약
- 전체 토핑/음료 선택 결과
- 총 라인 수와 총 금액
- `새 주문 시작` 버튼

## 5. 주문 흐름

주문 흐름은 `메뉴 선택 + 커스터마이징 + 장바구니 적재` 방식으로 고정한다.

1. 사용자가 좌측 메뉴 갤러리에서 메인 메뉴를 선택한다.
2. 우측 상단 패널이 해당 메뉴 기준으로 열린다.
3. 사용자는 토핑을 직접 선택하거나 추천 조합을 한 번에 적용한다.
4. 필요 시 음료를 추가하고 메인 메뉴 수량을 조절한다.
5. `장바구니 담기`를 누르면 현재 구성 전체가 하나의 라인 아이템으로 장바구니에 추가된다.
6. 같은 메뉴라도 토핑이나 음료가 다르면 다른 라인 아이템으로 분리된다.
7. 장바구니에서 메뉴별 삭제 버튼으로 개별 취소할 수 있다.
8. `주문하기`를 누르면 완료 화면이 보이고, 주문 기준으로 추천 통계가 반영된다.
9. `새 주문 시작`을 누르면 초기 상태로 복귀한다.

## 6. 추천 기능 설계

### 목표

추천 기능은 백엔드 없이도 `실시간으로 인기 조합을 추천받는 느낌`을 주어야 하며, 멀티카트 구조에서도 메뉴 단위 추천이 정확하게 유지되어야 한다.

### 추천 방식

추천은 두 데이터 소스를 합산해 계산한다.

- 기본 인기 조합 데이터
- 현재 세션 및 브라우저 저장 데이터

초기 상태에서도 추천이 비어 보이지 않도록, 메뉴별 대표 조합의 초기 점수를 미리 넣는다.

예시:

- 기본 야끼소바: `치즈 + 계란 프라이 추가`
- 기본 야끼소바: `비엔나 + 치즈`
- 소금 야끼소바: `오징어새우 + 면 추가`
- 소금 야끼소바: `오징어새우 + 계란 프라이 추가`

### 추천 로직

- 메인 메뉴별로 토핑 조합 통계를 관리한다
- 메뉴 선택 시 해당 메뉴 기준 상위 추천 조합 2~3개를 노출한다
- 추천 카드를 누르면 해당 토핑이 한 번에 선택된다
- 장바구니 담기 이벤트는 현재 세션의 보조 통계로만 기록하고, 화면 랭킹 계산에는 반영하지 않는다
- 최종 통계 점수 증가는 `주문하기` 완료 시점 기준으로 처리한다
- 통계는 `localStorage`에 저장해 새로고침 후에도 유지한다

### 추천 표시 방식

추천 카드는 단순 텍스트가 아니라 아래 정보를 담는다.

- `지금 가장 많이 담는 조합` 문구
- 토핑 조합명
- 추천 배지 또는 랭킹 표시
- 한 번에 적용 가능한 CTA
- 입체적으로 떠 있는 작은 카드형 3D 표현

### 데이터 복구

- 저장된 통계 데이터가 없으면 기본 인기 조합 데이터로 시작한다
- 저장된 데이터 형식이 손상되었으면 기본 데이터로 안전하게 복구한다
- 추천 결과가 비어 있으면 `가장 먼저 인기 조합을 만들어보세요` 문구를 보여준다

## 7. 상태 모델

프론트엔드 상태는 아래 단위로 나눈다.

- `menuCatalog`: 메뉴, 가격, 설명, 이미지, 카테고리 정보
- `toppingCatalog`: 토핑명과 가격 정보
- `drinkCatalog`: 음료명과 가격 정보
- `selectedMainMenu`: 현재 커스터마이징 중인 메인 메뉴
- `selectedToppings`: 현재 선택된 토핑 목록
- `selectedDrinks`: 현재 메뉴에 연결된 음료별 수량 목록
- `selectedQuantity`: 현재 커스터마이징 중인 메인 메뉴 수량
- `cartItems`: 장바구니에 담긴 메뉴 세트 라인 아이템 배열
- `cartSummary`: 모든 장바구니 라인 아이템의 총액과 개수를 계산한 요약 정보
- `comboStats`: 메뉴별 토핑 조합 통계

각 `cartItems` 항목은 아래 정보를 가진다.

- 메인 메뉴 식별자
- 메인 메뉴명
- 선택 토핑 목록
- 선택 음료 목록과 수량
- 메인 메뉴 수량
- 라인 총액
- 삭제용 고유 식별자

이 구조는 커스터마이징 중 상태와 장바구니에 이미 담긴 확정 상태를 명확히 분리하기 위해 사용한다.

## 8. 인터랙션 설계

### 메뉴 선택

- 메인 메뉴 카드 선택 시 활성 상태가 즉시 반영된다
- 우측 상단 패널 상단 정보와 추천 카드가 해당 메뉴 기준으로 다시 그려진다
- 메뉴를 변경하면 이전 메뉴의 토핑, 음료, 수량 선택은 기본값으로 초기화한다
- 장바구니에 이미 담긴 항목은 유지된다

### 토핑 선택

- 토핑은 큰 버튼형 칩 UI로 표시한다
- 선택 시 색상, 그림자, 살짝 솟아오르는 효과로 활성 상태를 보여준다
- 선택/해제할 때 현재 구성 소계가 즉시 갱신된다

### 추천 카드 적용

- 추천 카드 클릭 시 해당 토핑 조합이 한 번에 적용된다
- 이미 일부 토핑이 선택된 상태여도 추천 카드 기준으로 현재 토핑 상태를 교체한다
- 추천 적용 직후 현재 구성 소계와 요약 영역을 다시 계산한다

### 음료 선택

- 음료는 현재 선택 메뉴에 연결되는 옵션으로 취급한다
- 여러 종류를 동시에 담을 수 있다
- 각 음료는 독립적으로 추가/감소할 수 있다
- 음료 가격은 현재 메뉴 구성 소계에 즉시 반영된다

### 장바구니 담기

- `장바구니 담기`를 누르면 현재 메뉴 구성 전체가 하나의 라인 아이템으로 추가된다
- 같은 메뉴라도 토핑이나 음료가 다르면 별도 라인으로 추가된다
- 추가 직후 현재 커스터마이징 상태는 마지막 선택값을 유지해 같은 메뉴를 빠르게 재주문할 수 있게 한다

### 장바구니 삭제

- 각 장바구니 라인에는 메뉴별 삭제 버튼이 있다
- 삭제 시 해당 라인만 제거되고 다른 라인은 유지된다
- 삭제 직후 장바구니 총액과 총 개수가 즉시 재계산된다

### 주문 완료

- 주문 완료 시 장바구니 전체 내용을 완료 화면에 표시한다
- 동시에 각 메인 메뉴에 대한 추천 통계 데이터를 반영한다
- 완료 후 `새 주문 시작`으로 초기 화면 복귀가 가능해야 한다

## 9. 예외 처리

- 메인 메뉴를 선택하지 않으면 `장바구니 담기` 버튼을 비활성화한다
- 장바구니가 비어 있으면 `주문하기` 버튼을 비활성화한다
- 장바구니에서 메뉴별 삭제를 눌러도 다른 라인이 영향을 받지 않아야 한다
- 같은 메뉴를 다른 조합으로 여러 번 담아도 라인 아이템이 서로 덮어써지지 않아야 한다
- 실사 사진이 누락되거나 로드 실패해도 카드 전체가 깨지지 않아야 한다
- 추천 데이터가 없거나 깨져도 화면 전체가 실패하지 않아야 한다
- `localStorage` 사용이 불가능한 환경에서도 기본 추천 데이터만으로 동작 가능해야 한다

## 10. 반응형 원칙

- 데스크톱/키오스크 가로 화면에서는 좌측 메뉴, 우측 상단 커스터마이징, 우측 하단 장바구니 구조 유지
- 태블릿에서는 우측 패널의 위아래 분할 비율을 조정해 장바구니가 항상 보이도록 한다
- 모바일 폭에서는 메뉴 갤러리, 커스터마이징 패널, 장바구니 패널 순으로 세로 스택 전환
- 3D 효과가 작은 화면에서 과하지 않도록 그림자와 블러 강도를 축소한다
- 모든 주요 버튼과 칩은 터치 친화적인 크기로 유지한다

## 11. 구현 범위

이번 구현 범위는 아래까지 포함한다.

- 브랜드형 메인 키오스크 화면
- 실제 음식 사진 카드 UI
- 소프트 3D 스타일의 카드, 패널, 버튼 디자인
- 우측 상단 커스터마이징 패널 UI
- 우측 하단 장바구니 UI
- 토핑/음료 선택 및 가격 계산
- 장바구니 담기와 메뉴별 삭제
- 추천 조합 계산 및 적용
- `localStorage` 기반 추천 데이터 유지
- 주문 완료 화면
- 전체 비우기/새 주문 시작 흐름
- 반응형 스타일

이번 구현 범위에서 제외한다.

- 실제 결제 연동
- 서버 저장소
- 관리자 화면
- 주문 이력 백오피스

## 12. 테스트 기준

다음 동작은 구현 후 반드시 검증한다.

- 실사 음식 사진이 정상 표시되고, 로드 실패 시 폴백 UI가 보이는지
- 메인 메뉴 선택 시 우측 상단 커스터마이징 패널이 올바르게 갱신되는지
- 토핑 추가/해제 시 현재 구성 소계가 즉시 재계산되는지
- 추천 카드 클릭 시 토핑이 일괄 적용되는지
- 서로 다른 메뉴를 여러 개 장바구니에 담을 수 있는지
- 같은 메뉴를 다른 조합으로 여러 번 담아도 별도 라인으로 쌓이는지
- 각 메뉴 라인을 개별 삭제해도 총액이 정확히 갱신되는지
- 주문 완료 후 각 메뉴의 추천 조합 통계가 증가하는지
- 저장된 추천 데이터가 있어도 새로고침 후 정상 반영되는지
- 모바일, 태블릿, 큰 화면에서 레이아웃이 무너지지 않는지
- 3D 스타일이 적용되어도 텍스트 가독성과 조작성이 유지되는지

## 13. 구현 메모

- 기존 프로젝트는 정적 파일 구조이므로 `index.html`, `css/style.css`, `js/app.js` 중심으로 재구성한다
- 추천 계산과 상태 관리는 별도 데이터/로직 레이어로 분리해 DOM 조작 코드와 섞이지 않게 한다
- 메뉴/토핑/음료 데이터는 상수 형태로 분리해 이후 실제 이미지와 문구 교체를 쉽게 만든다
- 공개 사용 가능한 실제 이미지의 라이선스와 출처 정보를 정리해 프로젝트 내부 에셋으로 보관한다
- 3D 효과는 과한 회전보다 그림자, 하이라이트, 깊이 계층으로 구현해 키오스크 사용성을 해치지 않게 한다
